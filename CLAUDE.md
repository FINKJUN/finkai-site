# FINKAI Website — Project Guide for Claude

This file is the authoritative reference for making changes to the FINKAI website (finkai.co). Read this fully before touching any file.

## 1. What This Project Is

**finkai.co** is the marketing website for FINKAI — an AI-powered financial intelligence platform that connects to ERP systems (SAP, Oracle, TallyPrime, QuickBooks) and answers financial questions in plain English. Answers are computed arithmetically from live ERP data, not generated by a language model.

**Repo:** `https://github.com/FINKJUN/finkai-site`
**Deployment:** Netlify (auto-deploys on push to `master`)
**Live URL:** finkai.co

### Carousel system
```js
mcNext('carousel-id')   // next slide
mcPrev('carousel-id')   // prev slide
```
Carousel HTML: `.mc-wrap#id > .mc-slides > .mc-slide.active` + `.mc-nav`
The JS (`mcNav` function) is generic — works with any `id`.

### Modal system
```js
fkOpen('fk-[name]-modal')    // opens modal (function derives overlay id)
fkClose('fk-[name]-overlay') // closes overlay
fkBgClose(event, 'fk-[name]-overlay') // close on background click
```
Modal HTML: `.fk-overlay#fk-[name]-overlay > .fk-modal#fk-[name]-modal`
ESC key closes all open modals automatically.

## 5. Content Rules & Tone

### Voice
- Direct, confident, no fluff
- Speak to a 45-year-old CFO — credible, specific, numbers-driven
- No superlatives without evidence ("best", "only", "revolutionary")
- Short sentences. Active voice.

### Positioning
- **Global-first** — do not position as India-only
- **Keep** as features: SEBI LODR, Ind AS, GST, TDS, Schedule III (genuine compliance features, not location locks)
- **Remove/avoid** in marketing copy: "Built for Indian CFOs", "India's platform", "Indian finance teams", "for India"
- Customers are Ananta Group customers — always add this context

### Accurate AI framing — non-negotiable
FINKAI uses LLMs for natural language understanding but computes answers arithmetically. Never say:
- ❌ "Not a language model"
- ❌ "Zero hallucination"
- ❌ "Does not use AI"

Always say:
- ✅ "AI understands your question. Arithmetic answers it."
- ✅ "Reduced hallucination risk"
- ✅ "Uses AI to understand the question, then computes the answer from live ERP data"

### Banned names / content
- ❌ **Madurai Power Corp** — must never appear anywhere
- ❌ **SPI** or **SPI Group** — must never appear anywhere
- ❌ **RJ Narayanan** — not part of FINKAI
- ❌ "Series A", funding stage, valuation — not confirmed, do not include
- ❌ "by FINAHQ" next to FINKAI name

## 10. How to Make Incremental Changes

### Adding a new section
Find the right insertion point using the section order above. Use Python scripts for large injections to avoid Edit tool character limits.

### Replacing a mockup
Use the `replace_mockup(content, label, new_html)` pattern:
```python
def replace_mockup(content, label, new_html):
    idx = content.find(label)
    start = content.rfind('<div class="mockup">', 0, idx)
    pos = start + len('<div class="mockup">')
    depth = 1; end = pos
    while depth > 0 and pos < len(content):
        no = content.find('<div', pos)
        nc = content.find('</div>', pos)
        if nc == -1: break
        if no != -1 and no < nc: depth += 1; pos = no + 4
        else: depth -= 1; end = nc + 6; pos = nc + 6
    return content[:start] + new_html + content[end:]
```

### Running scripts on Windows
```bash
python script.py   # NOT python3
```

### Checking for broken links before committing
```bash
grep -oh 'href="[^"]*"' index.html | sort -u
```

### Committing
```bash
git add index.html  # stage specific files, not git add -A
git commit -m "Description"
git push origin master
```
